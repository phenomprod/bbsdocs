2 [ MOVE ] - Move file(s)

This command allows SysOp to move files from conference/catalog to the another. Wildcards are supported. Should be easy to figure out how this one works.

3 [ LINK ] - Link file(s)

Mostly the same as MOVE, but instead of moving creates a symlink.

4 [ COPY ] - Copy file(s)

Mostly the same as MOVE, but instead of moving copies the file.

5 [ SD ] - SysOp download

This command allows SysOp to download any file from the harddisk. File won't be marked to the download bytes/files.

6 [ LOG ] - View logfile

This command allows SysOp to view logfile of the specified node.

7 [ $ ] - Test door

This command allows SysOp to test a door program before installing it. The first parameter is the pathname of the tested door and the remaining parameter, if any, are just the normal parameters for the door command.

Note that you can not specify the command line arguments with which the door executable is exec'd. The only argument provided to the door is the node number of the current node. For most doors, this is OK.

8 [ LU ] - Local upload

Upload file(s) locally. DD prompts you to copy files to a temp dir and after you have copied the files, hit return and DD puts the files online.
